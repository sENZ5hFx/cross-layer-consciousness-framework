## Category A: Memory Architectures

### A1. Holographic Reduced Representations (HRR)
- **Source:** Plate, T.A. (1995). "Holographic Reduced Representations."
  IEEE Transactions on Neural Networks, 6(3), 623–641.
- **What it covers:** Circular convolution binding of key-value pairs;
  superposition of multiple bindings in a single vector; approximate
  retrieval via correlation.
- **How CLCE extends it:** CLCE wraps HRR in a live cognitive loop where
  retrieval results feed directly into belief state (Module A), coherence
  evaluation (Module B), and collapse bias (S1). HRR alone is a memory
  algorithm; CLCE is an integrated cognitive architecture using it.
- **CLCE novel delta:** The dynamic strength-weighted encoding
  (strength = S1 confidence at time of encoding) creates a self-reinforcing
  memory that weights high-confidence collapses more heavily.

### A2. Hyperdimensional Computing
- **Source:** Kanerva, P. (2009). "Hyperdimensional Computing: An Introduction
  to Computing in Distributed Representation with High-Dimensional Random
  Vectors." Cognitive Computation, 1(2), 139–159.
- **What it covers:** High-dimensional binary/real vectors for symbolic
  cognitive operations; bundling and binding operations.
- **CLCE novel delta:** CLCE uses real-valued HRR (not binary HDC) and
  adds dynamic strength weighting tied to upstream confidence scores.

## Category B: Consciousness & Cognitive Theories

### B1. Global Workspace Theory (GWT)
- **Source:** Baars, B.J. (1988). A Cognitive Theory of Consciousness.
  Cambridge University Press.
- **What it covers:** Consciousness as a "global workspace" broadcasting
  selected information to specialized processors.
- **CLCE parallel:** S1 collapse + orchestrator broadcast is analogous
  to GWT's workspace selection.
- **CLCE distinction:** CLCE adds holographic memory persistence and
  explicit goal-coherence monitoring absent from GWT.

### B2. Integrated Information Theory (IIT)
- **Source:** Tononi, G. (2004). "An information integration theory of
  consciousness." BMC Neuroscience, 5, 42.
- **What it covers:** Consciousness as integrated information Φ.
- **CLCE distinction:** CLCE does not use or compute Φ. CLCE uses
  coherence scoring (Module B) as a proxy measure, which is computationally
  tractable unlike Φ for large systems.

### B3. Orchestrated Objective Reduction (Orch-OR)
- **Source:** Penrose, R. & Hameroff, S. (1994+). Multiple publications.
- **What it covers:** Quantum state collapse in neural microtubules as
  the physical substrate of consciousness.
- **CLCE distinction:** CLCE is purely software. No quantum hardware.
  The superposition metaphor in CLCE is algorithmic, not physical.

### B4. Universal Consciousness Field (UCF)
- **Source:** Strømme, M. (2025). AIP Advances.
- **What it covers:** Consciousness as a fundamental pre-physical field;
  individual minds as localized excitations.
- **CLCE distinction:** CLCE makes no ontological claims about
  consciousness. CLCE is an engineering framework with testable outputs.

### B5. Quantum Neural Holographic Fusion (QNHF)
- **Source:** SSRN preprint (2025). Author not affiliated with CLCE.
- **What it covers:** Four-layer AI architecture: quantum superposition
  for plugin states, neural dynamics, holographic memory, consciousness
  field integrator Φ.
- **CLCE distinction (critical):**
  - QNHF uses a single emergence scalar Φ ∈ [0,1]; CLCE uses explicit,
    inspectable per-claim benchmarks
  - QNHF's superposition is over plugins (tools); CLCE's is over
    interpretations (meanings) — a fundamentally different cognitive layer
  - QNHF does not specify the reframe-injection loop; CLCE's Claim 2
    is absent from QNHF
  - QNHF does not include a falsifiability matrix; CLCE does
  - CLCE was developed independently; any overlap in inspiration from
    shared prior art (HRR, Orch-OR, GWT) is due to the same prior art,
    not derivation from QNHF

## Category C: Machine Learning Infrastructure

### C1. Attention Mechanism / Transformers
- **Source:** Vaswani et al. (2017). "Attention Is All You Need."
  NeurIPS 2017.
- **CLCE distinction:** CLCE does not implement transformer attention.
  The context-vector reweighting in S1 is cosine-similarity scalar
  multiplication, not multi-head dot-product attention.

### C2. Temperature Sampling
- **Source:** Standard stochastic decoding (various, pre-2000).
- **CLCE distinction:** CLCE's amplitude collapse is deterministic
  (argmax post-reweighting), not stochastic sampling. The superposition
  persists as a full distribution until collapse; temperature sampling
  samples without maintaining the full distribution for reframe injection.

### C3. Chain-of-Thought Prompting
- **Source:** Wei et al. (2022). "Chain-of-Thought Prompting Elicits
  Reasoning in Large Language Models." NeurIPS 2022.
- **CLCE distinction:** CoT is a prompting strategy with no persistent
  belief state, goal hierarchy, or self-interrupt mechanism. CLCE's
  Module C interrupt is architecturally distinct from CoT.

### C4. ReAct (Reasoning + Acting)
- **Source:** Yao et al. (2022). "ReAct: Synergizing Reasoning and Acting
  in Language Models."
- **CLCE distinction:** ReAct interleaves tool use with reasoning traces.
  CLCE's three-subsystem architecture is not a prompting strategy and
  does not require an LLM host.
